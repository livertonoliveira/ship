Pipeline de Desenvolvimento (/ship:run)

Includes lazy @@ship/...

Modos de Armazenamento

Determinado por ship/config.md → Linear Integration → Configured

🔗 Linear Mode

Specs viram Documents; tasks viram Issues com milestones/labels; quality reports viram Comments; drift viram sub-issues. Zero arquivos locais além de ship/config.md.

💾 Local Mode

ship/changes/<feature>/ guarda proposal.md, design.md, tasks.md, plan.md, report-<task>.md, tracking.md, drift-report.md.

Compartilhado (independe do modo)

ship/config.md

Sempre local — config do projeto, gates, test scope, convenções.

.context/ship-run/<task-id>/

Scratch dir efêmero por execução: stack.md, diff.md, phase-status.md, dispatch-log.md, drift-findings.json.

ship/audits/<tipo>-<data>.md

Relatórios de auditoria project-wide (persistem sempre em disco).

ship/config.md

Único artefato sempre local — controla comportamento de todo o pipeline
